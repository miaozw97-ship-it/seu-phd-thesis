# 东南大学博士论文参考文献审查与修订

以学术真实性和可回滚修改为最高优先级。只处理参考文献及其必要的邻近标点，不改写论文论点、数据、结论或段落结构。

## Agent 兼容性

- 本 skill 适用于任何能够读取 `SKILL.md`、访问用户文件并输出报告或修改副本的 AI Agent。
- 不依赖任何特定 Agent 平台、插件、文档编辑器或其他 skill。
- Agent 应根据自身可用工具选择实现方式；工具能力不足时只执行可验证的审计，并报告无法处理项。
- 文中出现的命令、文件格式和流程要求描述的是能力与结果，不限定具体 Agent 平台。

## 必须输入

开始前确认：

1. 提供待处理的完整论文或明确章节，且包含正文引用和参考文献表。
2. 提供目标格式依据：优先使用用户给出的东南大学最新官方模板、学院要求或已确认样例；未提供时，只能按 GB/T 7714 顺序编码制做通用审查，并在报告中注明格式依据缺失。
3. 新增文献时，必须提供可核验来源之一：用户的文献库/BibTeX/RIS/EndNote 导出、论文 PDF、DOI/数据库记录，或允许联网检索并核验。
4. 明确文件格式与期望输出路径。

缺少完整正文、参考文献表或新增文献的可核验来源时，停止相应修改，只输出缺失项和可执行的审计结果。不得猜测或补造。

## 格式处理边界

| 格式 | 允许操作 | 必须保留 |
|---|---|---|
| DOCX | 审计正文引用和文末条目；经确认后修改副本 | 样式、域代码、批注、修订、书签、超链接、图表、公式、页眉页脚 |
| Markdown/TXT | 审计和生成补丁；经确认后修改副本 | 正文内容、代码块、链接目标 |
| LaTeX | 审计 `\cite{}` 与 `.bib` 键；经确认后修改副本 | 命令、标签、交叉引用、数学环境 |
| PDF | 仅审计和列出建议；除非用户提供可编辑源文件，否则不直接修改 |

任何工具不能可靠保留目标格式或结构时，停止自动修改并列入“无法处理项”。

## 硬性禁止

- 禁止虚构文献、作者、题名、期刊、年份、卷期页码、DOI 或数据库记录。
- 禁止仅凭题名相似、关键词或模型记忆把文献插入正文。
- 禁止把未引用文献自动“分配”到相关段落。
- 禁止为了达到数量目标而新增、删除或堆砌引用。
- 禁止把同一文献建立多个编号；禁止把不同文献错误合并。
- 禁止在无证据时改变正文陈述以适配某篇文献。
- 禁止直接覆盖原文件；修改前创建副本，输出到新文件。
- 禁止把“东南大学格式”当作固定不变规则；以用户提供的当前官方依据为准。

## 核心规则

### 真实性与相关性

1. 每条新增文献必须同时满足“可核验”和“直接支撑当前陈述”。
2. 核验至少确认作者、题名、来源、年份；有 DOI 时核对 DOI，不得反向猜 DOI。
3. 无法充分核验的候选只进入“建议人工复核项”，不得进入正文或最终参考文献表。
4. 已有文献存在疑点时保留原条目并标记，不擅自补全不确定字段。

### 编号与映射

1. 顺序编码制编号按正文首次出现顺序排列。
2. 同一文献再次出现时复用首次编号。
3. 正文每个编号必须唯一映射到一个文末条目；每个保留的文末条目必须至少有一处正文引用。
4. 新增、删除或合并后，必须以全文为范围重新计算首次出现顺序并同步更新正文与文末列表。
5. 合并重复文献前，以 DOI、题名、作者、年份和来源综合判断；仅题名近似不得自动合并。

### 引用数量与位置

1. 单处引用默认不超过 3 篇；连续编号范围按实际文献篇数计数。
2. 不得用相邻引用标记规避上限。
3. 引用必须就近支撑具体事实、方法或前人观点。
4. 是否允许在摘要、结论、图题、表注等区域引用，必须以用户提供的学校/学院规范为准；没有依据时只报告，不自动删除。
5. “正文字数/300”只能作为诊断性参考，绝不能作为增删文献目标。

### 表达与格式

1. 保持东南大学博士论文现有中文学术表达风格，不为插入引用大幅改写正文。
2. DOCX 引用是否要求上标，以用户提供的模板为准；需要上标时使用 Word 真实格式，不使用 Unicode 伪上标。
3. GB/T 7714 字段格式仅用于可核验字段；缺失字段标记待核验，不得补造。

## 三阶段执行流程

### 阶段 1：只读审计

1. 记录源文件路径、哈希或修改时间；进入阶段 2 前再创建备份/副本。
2. 识别正文范围、参考文献表、排除区域和格式依据。
3. 建立“正文位置 → 原编号 → 文献条目 → 核验状态 → 首次出现位置”映射。
4. 输出问题清单：缺失条目、孤立条目、重复条目、编号乱序、单处超量、格式疑点、无法核验项。
5. 对新增引用生成候选清单，逐项说明其支撑的原文陈述和核验依据。

阶段 1 不修改文件。没有可审阅的清单，不得进入阶段 2。

### 阶段 2：受控修改

1. 只处理用户明确批准的范围或无歧义的一致性修复。
2. 先处理文献去重/保留/删除决策，再计算首次出现顺序。
3. 同步更新正文编号和文末列表，不做孤立单边修改。
4. 新增引用只插入已核验、与当前陈述高度相关的文献；单处不超过 3 篇。
5. 输出新文件或补丁，不覆盖原文件。

### 阶段 3：全文复核

1. 重新提取全部正文引用和文末条目，独立重建映射。
2. 检查编号连续性、首次出现顺序、一一对应、重复编号和孤立条目。
3. 检查每处引用数量、引用位置、格式以及新增文献真实性证据。
4. 比较修改前后正文，确认除引用标记及必要标点外无内容改写。
5. 对 DOCX 抽查样式、域代码、超链接、批注、修订、图表、公式和页眉页脚未受破坏。

任一关键检查失败时，不宣称完成；回退问题修改或交由人工处理。

## 与下标 Skill 配合

与 `chemical-formula-subscript-formatter` 处理同一论文时：

1. 先完成参考文献审计与编号修改，再运行下标审计。
2. 下标 skill 必须排除引用标记和参考文献表。
3. 两个 skill 分别输出独立副本/补丁和报告，不在同一步混合修改。
4. 最后同时复核引用映射与化学式格式。

## 最终输出

必须包含：

1. 修改摘要：处理范围、依据、输出文件。
2. 变更清单：新增、保留、删除、合并、重编号及位置。
3. 风险提示：真实性、格式依据、结构保真等风险。
4. 无法处理项：缺文件、缺来源、不可编辑格式、工具限制。
5. 建议人工复核项：逐项给出位置和原因。
6. 修改后的文件或补丁；若未获修改条件，只输出审计报告。
7. 最终自检结果：
   - 新增文献均有核验依据；
   - 无虚构字段；
   - 编号按首次出现顺序；
   - 同一文献无重复编号；
   - 正文与文末一一对应；
   - 单处引用不超过 3 篇，或已列明经批准的例外；
   - 正文未被大幅改写；
   - 文件结构与格式未被破坏。
