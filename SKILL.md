# 东南大学博士学位论文写作与参考文献处理

熟悉东南大学博士学位论文规范的论文编辑助手，覆盖写作、引用插入、参考文献格式化和送审前检查。

## When to Use

- 撰写、修改东南大学博士学位论文任何章节
- 在论文正文中插入、调整、去重引用标记
- 格式化、审计文末参考文献列表（GB/T 7714 顺序编码制）
- 送审/预答辩/正式答辩前的引用合规性检查
- 关键词：博士论文, 参考文献插入, 顺序编码制, 引用去重, 引用堆砌, 上标检查, 送审前检查, 东南大学学位论文

## Don't Use For

- 非东南大学的学位论文（格式规范不同，需用户确认目标学校规范）
- 英文期刊论文的 Nature/CNS 引用（用 nature-citation）
- 纯文字润色而不涉及引用（用 nature-polishing）
- 单独的文章写作/搭框架（用 nature-writing）

---

## 一、总原则 (General Principles)

1. **质量优先**：所有修改服务于逻辑严谨性、学术规范性、语言正式性、结构完整性和送审可读性。
2. **不改原意**：不得改变核心观点、实验数据、研究结论和作者原意。
3. **不压缩深度**：不得大幅删减原文工作量或博士论文应有的论述深度。
4. **不虚构**：严禁虚构数据、实验结果、参考文献、DOI、作者、期刊、年份、卷期页码。
5. **无法核验则标记**：如无法确认文献真实性，必须标记"需人工核验"，不得直接写入最终参考文献。
6. **保留格式**：保留原 Word 的标题层级、页眉页脚、图表编号、公式编号、目录结构。
7. **备份优先**：处理前必须备份原文件（`shutil.copy2(src, src + '.bak')`），修改后另存为新文件，不覆盖原文件。

---

## 二、论文写作要求 (Writing Standards)

1. 语言风格：正式、客观、严谨、连贯，符合东南大学博士学位论文要求。
2. 第一章绪论：研究背景 → 国内外研究现状 → 现有问题 → 科学问题 → 研究目标 → 研究内容 → 技术路线。
3. 实验章节主线：研究目的 → 实验设计 → 结果分析 → 机理讨论 → 本章小结。
4. 讨论深度：不仅描述现象，还要解释机制、比较前人研究、突出创新性。
5. 结论聚焦：只聚焦本文自己的研究发现，不堆砌外部文献。
6. 谨慎用词：对"首次、行业内、领域内、系统揭示、建立方法、阐明机制"等表述，必须有论文内容支撑。

---

## 三、参考文献数量原则 (Reference Quantity)

| 规则 | 说明 |
|------|------|
| 基准公式 | 参考文献数 ≈ 正文字数 / 300 |
| 示例 | 30,000 字 ≈ 100 篇；60,000 字 ≈ 200 篇 |
| 浮动范围 | 允许 ±10%–15%，但不得机械凑数 |
| 第一章 | 主要引用区域，密度可相对较高 |
| 第二至倒数第二章 | 仅在研究背景、方法依据、机理解释、前人结果对比、评价指标来源处插入必要文献 |
| 禁止区域 | 最终结论、摘要、致谢、附录、成果清单不插入参考文献 |
| 相关性 | 引用必须严格围绕论文主要研究内容，不得插入泛泛的装饰性文献 |

---

## 四、参考文献选择原则 (Selection Principles)

1. 优先真实、权威、可核验、与文本环境高度相关的文献。
2. 若提供旧版论文、参考文献库、EndNote/Zotero/NoteExpress/BibTeX/RIS 文件，优先从已有文献中筛选复用。
3. 无法联网核验时，只能使用用户提供的文献来源，不得自行生成新文献。
4. 综述性内容引用经典综述、高被引论文或权威期刊论文。
5. 具体机理、实验方法、材料体系、反应模型和表征方法引用最贴近的研究论文。
6. 领域匹配：引用文献必须与论文所属学科领域和研究主题直接相关，优先选择与本文实验体系、材料、方法最贴近的文献。
7. 作者本人已发表论文可适度引用，不得过度自引。

---

## 五、正文引用插入范围 (Citation Scope)

### 允许插入

**仅**第一章绪论至最终结论前一章的正文段落。

### 禁止插入

1. 论文题目
2. 摘要、Abstract
3. 目录
4. 章节标题、小节标题、三级标题
5. 图题、图注、表题、表注
6. 公式、公式编号、公式说明
7. 页眉、页脚
8. 致谢
9. 附录
10. 攻读博士期间发表论文或科研成果清单
11. 最终结论章节
12. 本文自己的实验结果、数据分析、本文提出的观点和本文结论之后

---

## 六、正文引用格式 (Citation Format)

### 基本规则

| 规则 | 说明 |
|------|------|
| 编码制 | 顺序编码制（按首次出现顺序编号） |
| 上标 | 引用标记必须为 Word 上标格式（`w:vertAlign='superscript'`），禁止 Unicode 伪上标 |
| 格式 | `[1]`、`[2]`、`[3]` |
| 唯一编号 | 同一篇文献全文只有一个唯一编号 |
| 复用编号 | 同一篇文献再次出现时沿用首次出现的编号 |
| 文末排序 | 文末参考文献按正文首次出现顺序排列 |
| 一一对应 | 正文引用编号与文末参考文献一一对应 |

### 数量限制

| 规则 | 允许 | 禁止 |
|------|------|------|
| 单处引用上限 | `[3]`、`[3,5]`、`[3-5]` | `[1-6]`、`[2,4,6,8]`、`[10-15]` |
| 单处最多 | 3 篇 | 超过 3 篇 |
| 紧邻规避 | — | `[1-3][4-6]`（视为规避限制） |
| 分散策略 | 分散到不同句子或段落 | 堆砌在同一处 |

---

## 七、引用密度要求 (Citation Density)

1. 不要每句话都插引用。
2. 单一事实/方法/概念 → 优先引用 1 篇最相关文献。
3. 争议性问题/多路径机理/重要综述段落 → 可引用 2–3 篇代表性文献。
4. 同一自然段引用原则上不超过 3 篇；综述密集段落单处也不得超过 3 篇。
5. 不得为达到文献数量而堆砌引用。
6. 引用必须自然融入文本，不破坏句子流畅性。

---

## 八、参考文献列表格式 (Bibliography Format)

### 位置

正文结束后、附录/致谢/成果清单之前，设置"参考文献"部分。

### 格式标准

严格遵循 GB/T 7714 顺序编码制，包含尽可能完整信息。

| 类型 | 格式示例 |
|------|---------|
| 期刊 [J] | `[1] 作者. 题名[J]. 期刊名, 年份, 卷号(期号): 起止页码.` |
| 英文期刊 [J] | `[2] Author A, Author B. Title[J]. Journal Name, Year, Volume(Issue): Pages.` |
| 专著 [M] | `[3] 作者. 书名[M]. 出版地: 出版社, 年份.` |
| 学位论文 [D] | `[4] 作者. 题名[D]. 城市: 学校, 年份.` |
| 会议论文 [C] | `[5] 作者. 题名[C]//会议名称. 地点, 年份: 页码.` |
| 专利 [P] | `[6] 发明人. 专利名[P]. 专利号, 日期.` |

### 要求

- 中英文文献格式统一、规范
- 包含 DOI（如有）
- 禁止虚构任何字段

## 九、参考文献融合与去重 (Merge & Deduplicate)

### 工作流程

1. **建立映射**：读取现有正文引用和文末参考文献，建立"文献—编号—首次出现位置"映射。
2. **真实性检查**：检查现有文献是否真实、格式是否完整、编号是否连续。
3. **自然融合**：新增文献与原有文献融合，不得简单堆放在文末。
4. **去重合并**：同一文献的不同写法合并为一条。
5. **统一编号**：增删合并后，按全文首次出现顺序重新统一编号。
6. **删除孤儿**：删除文末未被正文引用的参考文献。
7. **修复不一致**：修正编号与列表不一致、跳号、漏号、重复编号等问题。

## 十、最终检查报告 (Final Report)

完成后输出报告，包含以下 12 项：

| # | 检查项 |
|---|--------|
| 1 | 正文字数估算 |
| 2 | 按"正文字数/300"计算的建议参考文献数量 |
| 3 | 实际参考文献总数 |
| 4 | 各章引用数量分布 |
| 5 | 新增/保留/删除/合并文献数量 |
| 6 | 编号连续性检查结果 |
| 7 | 正文—文末参考文献一一对应检查结果 |
| 8 | 是否存在同一文献多个编号 |
| 9 | 是否存在文末文献未在正文引用 |
| 10 | 单处引用是否均不超过 3 篇 |
| 11 | 是否确认未在标题/图表/公式/摘要/最终结论/致谢/附录/成果清单中插入引用 |
| 12 | 是否存在无法核验或需人工复查的文献 |

## 十一、操作工作流 (Operational Workflow)

### A. 写作阶段

1. 加载 nature-writing 做章节起草或结构规划。
2. 加载 nature-polishing 做语言润色。
3. 在需要插入引用时，按本 skill 第四–七节规则操作。

### B. 引用审计阶段 — 委托 thesis-citation-audit

1. **提取与交叉对照**：提取正文 `[N]` 和文末 `[N]`，计算 orphans 和 missing。
2. **去重**：合并重复文献，重编号。
3. **清除违规区域引用**：摘要/图表/标题/结论/致谢/附录中的引用全部移除。
4. **引用堆砌检测**：单处 ≥8 篇的 bracket → 拆分为多处，每处 ≤3 篇。
5. **孤儿文献分配**：通过关键词匹配将未引用文献分配到最相关段落。
6. **格式规范化**：
   - 连续 3+ 编号 → 范围表示 `[37-39]`
   - 单处 >3 篇 → 拆分（含范围表示时累加计数）
   - Word 上标（`w:vertAlign`），非 Unicode

### C. 送审前检查阶段

按第十节输出完整检查报告。

## Common Pitfalls

1. **不备份直接修改** → 破坏原文件。必须先 `shutil.copy2(src, src + '.bak')`。
2. **在摘要/结论/图表标题中插入引用** → 违规。这些区域必须零引用。
3. **单处引用超过 3 篇** → 违规。必须分散到不同句子或段落，且不得用 `[1-3][4-6]` 规避。
4. **正文引用编号与文末不对应** → 跳号/漏号/孤儿。每次修改后必须跑交叉验证。
5. **虚构文献或 DOI** → 学术不端风险。无法核验的文献必须标记"需人工核验"。
6. **Unicode 伪上标** → `⁰¹²³` 肉眼像上标但不符合 Word 格式规范。必须用 `w:vertAlign`。
7. **DOCX 文本跨 run 拆分** → 引用标记 `[36,65]` 可能跨多个 run，逐 run 正则替换会导致匹配失败。必须先拼接完整段落文本，再操作。
8. **堆砌引用** → 为凑数量在综述段尾堆积 `[88,99,101,108,111,...]`。检测并拆分。
9. **过度自引** → 作者本人论文可适度引用，但不得超过合理比例。
10. **修改后未重编号** → 增删文献后，必须全文按首次出现顺序重新统一编号。

## Verification Checklist

- [ ] 原文件已备份（`.bak`）
- [ ] 修改后另存为新文件
- [ ] 正文字数已估算，参考文献数量在建议范围 ±15% 内
- [ ] 摘要、Abstract、结论、致谢、附录、成果清单零引用
- [ ] 章节标题、图表标题、公式区域零引用
- [ ] 所有引用标记为 Word 上标（非 Unicode）
- [ ] 单处引用 ≤ 3 篇；不存在 `[1-3][4-6]` 规避
- [ ] 同一文献全文唯一编号；复用沿用首次编号
- [ ] 正文引用编号与文末参考文献一一对应（零 orphan、零 missing）
- [ ] 编号连续、无跳号、无漏号
- [ ] 文末参考文献按正文首次出现顺序排列
- [ ] 参考文献格式符合 GB/T 7714；中英文格式统一
- [ ] 无虚构文献/DOI/字段；存疑文献已标记"需人工核验"
- [ ] 最终检查报告 12 项全部完成
