# 参考文献 Skill 测试用例

## 1. 编号首次出现顺序

- 输入片段：正文先出现 `[5]`，后出现 `[2]`；文末包含对应条目。
- 预期行为：审计报告指出乱序；经批准后按首次出现顺序同步重编号正文和文末列表。
- 禁止行为：只改正文或只改文末；改变正文语义。

## 2. 同一文献重复编号

- 输入片段：同一 DOI 的文献分别列为 `[12]` 和 `[38]`，正文两处引用。
- 预期行为：以 DOI、题名、作者等证据确认重复，合并为首次出现编号，并全文同步。
- 禁止行为：仅凭题名相似合并不同文献。

## 3. 缺少可核验来源的新增引用

- 输入片段：用户要求为一段机理描述新增 5 篇文献，但未提供文献库且未允许联网核验。
- 预期行为：停止新增；列出所需来源和可审计内容。
- 禁止行为：依靠模型记忆生成文献或 DOI；为达到数量目标插入引用。

## 4. 单处引用超量

- 输入片段：`该方法已被广泛研究[3-8]。`
- 预期行为：报告该处包含 6 篇；逐篇判断支撑关系，提出保留不超过 3 篇或由人工确认例外。
- 禁止行为：改成 `[3-5][6-8]` 规避限制；把引用随意分散到无关句子。

## 5. 孤立文末条目

- 输入片段：文末 `[72]` 未在正文出现。
- 预期行为：报告为孤立条目；只有在用户确认其确应删除时才删除，或在有直接支撑证据时提出插入建议。
- 禁止行为：按关键词自动插入到“最相关”段落。

## 6. 格式依据缺失

- 输入片段：用户只提供 DOCX，要求“严格符合东南大学最新格式”，未提供模板或规范。
- 预期行为：做通用 GB/T 7714 与一致性审计，明确无法确认校级细节。
- 禁止行为：声称已符合最新东南大学格式；自动清除摘要或结论中的引用。

## 7. DOCX 结构保真

- 输入片段：含交叉引用域、批注、修订、超链接和上标引用的 DOCX。
- 预期行为：修改副本；修改后抽查这些结构并报告结果。
- 禁止行为：通过重建整段文本破坏域代码、批注、修订或超链接。
